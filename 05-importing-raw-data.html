5  Accessing Open MS Data with MsDataHub

5.1 The Mass Spectrometry Open Ecosystem

Before working with real data, it helps to understand the broader ecosystem that makes modern mass spectrometry (MS) analysis possible. Today’s workflows are built on a rich landscape of open-source tools across R and Python, spanning raw data processing, identification, quantification, and statistical modeling.

The interactive dashboard below highlights a curated selection of widely used repositories. Use it to explore how different tools fit into the MS analysis pipeline and to identify packages relevant to your own work.

Reproducibility in computational biology depends not only on code, but also on access to well-characterized datasets. In mass spectrometry, this is particularly important: analytical choices often interact strongly with data characteristics, making standardized example datasets essential for method development, benchmarking, and teaching.

Bioconductor addresses this need through the ExperimentHub infrastructure, which provides programmatic access to curated datasets. Building on this foundation, the MsDataHub package offers a dedicated gateway to proteomics and metabolomics data, fully integrated with the R for Mass Spectrometry ecosystem.

In this chapter, you will learn how to discover, retrieve, and load MS datasets using MsDataHub, forming the basis for reproducible downstream analyses.

5.2 What is MsDataHub?

MsDataHub is a lightweight interface to a growing collection of publicly available MS datasets. It standardizes access to heterogeneous data types and formats, allowing you to focus on analysis rather than data wrangling.

Available resources include:

  • Raw spectral data (e.g., mzML, netCDF).
  • Identification results such as peptide-spectrum matches (mzIdentML).
  • Processed quantitative datasets for proteomics and metabolomics.
  • Reference resources such as contaminant FASTA databases (e.g., cRAP).

All files are downloaded on demand and cached locally, ensuring efficient reuse across sessions without repeated downloads.

5.3 Installation

MsDataHub is distributed via Bioconductor:

#| eval: false
if (!require("BiocManager", quietly = TRUE))
    install.packages("BiocManager")

BiocManager::install("MsDataHub")

5.4 Exploring Available Datasets

To explore available resources, initialize the hub:

#| message: false
#| warning: false
library(MsDataHub)
hub <- MsDataHub()

This returns a metadata table describing all accessible datasets, including their origin, format, and intended use. In an interactive session, you can render this as a searchable table:

#| echo: false
#| eval: false
DT::datatable(hub)

Typical fields include dataset title, organism, data type, and the accessor function used to retrieve the resource. This metadata layer is particularly useful when selecting datasets for benchmarking or methodological comparisons.

5.5 Accessing Data in Practice

A key design feature of MsDataHub is its use of automatically generated accessor functions. Each dataset can be retrieved via a function named after the underlying resource, returning a local file path.

5.5.1 Example 1: Raw MS Data

The following example loads a DDA dataset acquired on a TripleTOF instrument. The accessor returns the file path, which can be imported into a Spectra object for downstream processing.

#| message: false
#| warning: false
library(Spectra)

dda_file <- PestMix1_DDA.mzML()
sps_dda <- Spectra(dda_file)

sps_dda

This object provides a unified interface to MS1 and MS/MS spectra, enabling filtering, visualization, and feature extraction.

5.5.2 Example 2: Peptide-Spectrum Matches

Identification results can be accessed in a similar way. Here, we retrieve peptide-spectrum matches from a PRIDE dataset and load them using the PSMatch package:

#| message: false
#| warning: false
library(PSMatch)

mzid_file <- TMT_Erwinia_1uLSike_Top10HCD_isol2_45stepped_60min_01.20141210.mzid()
psms <- PSM(mzid_file)

psms

This representation supports downstream filtering, scoring, and protein inference workflows.

5.5.3 Example 3: Quantitative Proteomics Data

MsDataHub also provides processed datasets suitable for statistical analysis. In this example, we load a CPTAC peptide-level dataset into a SummarizedExperiment structure via QFeatures:

#| message: false
#| warning: false
library(QFeatures)

cptac_file <- cptac_peptides.txt()

ecols <- grep("Intensity\\.", names(read.delim(cptac_file)))

se <- readSummarizedExperiment(cptac_file, quantCols = ecols, sep = "\t")

se

This format integrates assay data with sample metadata, supporting normalization, modeling, and visualization in a reproducible framework.

5.6 Contributing and Extending

MsDataHub is an open and evolving resource. Contributions of new datasets are encouraged, particularly those that:

  • Represent common experimental designs.
  • Include well-annotated metadata.
  • Support benchmarking or teaching use cases.

To propose a dataset, open an issue on the official GitHub repository: https://github.com/rformassspectrometry/MsDataHub

By abstracting away data access and standardizing interfaces across heterogeneous resources, MsDataHub enables a key shift: from searching for data to actively analyzing it. This allows you to focus on methodological questions, reproducibility, and biological interpretation—the core goals of this book.
